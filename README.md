# CS-C2170 Project - Getting Started

This document explains how to run the `my_lang` interpreter on Linux and gives a concise introduction to the language syntax.

## 1. Running the project (Docker on Linux)

### 1.1 Prerequisites

- Linux environment
- Docker installed and running

Check Docker installation:

```bash
docker --version
```

### 1.2 Build the Docker image

From the project root (`my_lang`):

```bash
docker build -t my_lang .
```
Note the `.` at the end of the command.

### 1.3 Run in interactive mode (REPL)

```bash
docker run -it my_lang
```

You should see:

```text
My Lang Interpreter
===================
>
```

Type an expression, press Enter, and the interpreter prints the inferred type and evaluated result.

Example:

```text
> 1 + 2
Int Int(3)
```

In this output, the first `Int` is the inferred type, and `Int(3)` is the evaluated value.

Exit the REPL with:

```text
exit
```

### 1.4 Run with an example code file

```bash
docker run -it my_lang examples/integers.stlc
```

This executes the file and opens the REPL.

### 1.5 Example file format

- One command/expression per line
- Empty lines are ignored
- Lines beginning with `//` are treated as comments (single-line comments)
- Nested imports are allowed with `import "other_file.txt"`

Example file content:

```text
let x: Int = 5
let y : Bool = let z : Int = 3 in x == z
// This is a comment
x
```

## 2. Language introduction

`my_lang` is a small statically typed functional language with:

- Integers and booleans
- Lambda functions and application
- `let` bindings
- Conditionals
- Recursion via `fix`
- Pairs, lists, and sum types
- Static type checking before evaluation



### 2.1 Terms (expressions)

Core term forms include:

- Variables: `x`
- Integers: `0`, `42`, `-7`
- Booleans: `true`, `false`
- Arithmetic: `t1 + t2`, `t1 - t2`, `t1 * t2`, `t1 / t2`
- Boolean ops: `t1 and t2`, `t1 or t2`, `not t`
- Equality: `t1 == t2` (only works for Int and Bool, otherwise result in Type error)
- Comparison: `t1 < t2` or `t1 > t2`
- If-then-else: `if t1 then t2 else t3`
- Functions: `fun x : T -> t`
- Application: `t1 t2`
- Let-in: `let x : T = t1 in t2`
- Fixpoint: `fix t`
- Pairs: `(t1, t2)`, `fst t`, `snd t`
- Lists: `nil [T]`, `cons h t`, `caselist s n c`, `reclist s n c`
- Sums: `inl t (T)`, `inr t (T)`, `casesum s l r`

### 2.2 Types

Supported types:

- `Int`
- `Bool`
- Function types: `(T1 -> T2)`
- Pair types: `(T1, T2)` (inferred from pair terms)
- List types: `[T]`
- Sum types: `(T1 + T2)`

Notes:

- Function and sum type constructors are parsed with parentheses, for example `(Int -> Int)` and `(Bool + Int)`.
- `nil` requires an explicit element type, for example `nil [Bool]`.

### 2.3 Declarations

In the REPL, top-level declarations use:

```text
let name : Type = expression
```

Example:

```text
let inc : (Int -> Int) = fun x : Int -> x + 1
```

After declaration, `inc` is available in later expressions.

Note that declarations and terms must be written on a single line.

## 3. Quick examples

### 3.1 Arithmetic and conditionals

```text
10 * (5 - 3)
if true then 42 else 0
```

### 3.2 Functions and let

```text
let f : (Int -> Int) = fun x : Int -> x * 2
f 21
```

### 3.3 Recursion (`fix`)

```text
let fact : (Int -> Int) = fix (fun self : (Int -> Int) -> fun n : Int -> if n == 0 then 1 else n * (self (n - 1)))
fact 5
```

### 3.4 Pairs, lists, sums

```text
fst (1, 2)
cons true (cons false (nil [Bool]))
casesum (inl true (Int)) (fun x : Bool -> x) (fun y : Int -> false)
```

## 4. Troubleshooting

- `Parse error: ...`: expression syntax is invalid.
- `Type error`: expression is syntactically valid but does not typecheck.
- `Error reading file: ...`: imported file path is invalid.
